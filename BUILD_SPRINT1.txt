3. Công cụ & Lệnh thực thi chuẩn hóa
Thay vì cài đặt rườm rà lên máy Windows, bạn đã chọn cách dùng Docker cực kỳ "chuẩn bài" của dân DevOps. Câu lệnh chuẩn để build độc lập bất kỳ service nào (bạn sẽ dùng lại nó cho file CI ở Sprint 2) là:

docker run --rm -v ${PWD}:/usr/src/app -v maven-cache:/root/.m2 -w /usr/src/app maven:3.9-eclipse-temurin-21 mvn clean package -pl tax -am -DskipTests

Với:
-pl <tên-service>: Chỉ đích danh service cần build.

-am (Also Make): Tự động dò tìm và build các module phụ thuộc.

Bind Mount (-v ${PWD}:/usr/src/app): Cầu nối giúp lấy code từ máy bạn đưa vào Docker build, sau đó trả kết quả (thư mục target chứa file .jar) ngược lại máy bạn.
